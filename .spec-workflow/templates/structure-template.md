## 代码组织原则

1. **单一职责**：每个文件应有一个明确的目的
2. **模块化**：代码应组织成可复用的模块
3. **可测试性**：构建易于测试的代码
4. **一致性**：遵循代码库中建立的模式

### 示例结构：
```
src/
└── dashboard/          # 自包含的仪表板子系统
    ├── server/        # 后端服务器组件
    ├── client/        # 前端资源
    ├── shared/        # 共享类型/工具
    └── public/        # 静态资源
```

### 关注点分离
- 仪表板与核心业务逻辑隔离
- 拥有独立的 CLI 入口点以独立运行
- 对主应用的依赖最小
- 可以在不影响核心功能的情况下禁用
